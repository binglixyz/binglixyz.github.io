Text normalization

Posted on 2017-12-05

Overview

There are 3 parts of text normalization, listed as follows:

  • Tokenization
    • Usage: Text -> Words
    • Example: “I love you” -> “I”, “love”, “you”
  • Lemmatization (includes stemming)
    • Usage: Words -> Lemma (stemming is just strip suffixes)
    • Example: “sings”,”sang”,”sung” -> “sing”
  • Sentence segmantation
    • Usage: Text -> Sentence
    • Example: “You are such a lovely girl. Come on. Dress on.” -> “You are such a lovely girl.”, “Come on.”, “Dress on.”
      They are applied as part of the normalization process:
  1. Tokenizing words from running text.
  2. Normalizing word formats.
  3. Sementing Sentence in running text .
    Noted that there are 2 set of running text. Therefore, we might need to iterate through the whole text several times.

Words and corpora

  • Corpus: a computer readable collection of text
  • Utterence
    • Disfluency
      • Fragment: Words like “main-mainly”
      • Filled pause: “uh”, “em”
    • Capitalize
  • Wordform: “sings”, “sang”
  • Measure of number of words
    • Word type: Types is the number of distinct words, denoted as $|V|$
    • Word token: Tokens is the total number of running words, denoted as $N$
    • Number of lemmas
  • Heap’s or Herdan’s law : $|V|$ = $kN^{\beta}$ What for?? Any usage of this law?

Word torkenization and normalization

  • Penn treebank tokenization
    • Usage: Seperating out $\underline{clitics}
    • Example: “doesn’t” -> “does”, “n’t”
  • Chose one wordfrom for multiple forms
    • Usage: For IR, we want the query use one word form and the backend could search for all wordforms; for IE, we want all wordfroms would be recognized and generate the information in one worform. My understanding, maybe a bit confused
  • Case folding
    • Usage: For speech recognization and IR, everything is mapped to lowercase; for some othertasks, case folding is generally not done.

Word segmentation in Chinese

  • MaxMatch algorithm

Lemmatization and Stemming

  • Porter Stemmer
    • Usage: based on series of rewrite $\underline{rules}, as a $\underline{cascade}, in which the output of each pass is fed as input to the next pass. The rules are human-designed.

Sentence segmentation

Generally focus on classifing the “.” symbol. Building a $\underline{binary classifier}$ that decides if the period is part of the word or is sentence boundary marker.

Reference

[1] Speech and Language Processing (3rd ed. draft), Dan Jurafsky and James H. Martin

Dump questions about pytorch

Posted on 2017-09-30

Usage of optimizer.zero_grad()

  • Why optimizer.zero_grad() before loss.backward() ?
    Ans: If we don’t zero the grads of optimizer, it will accumulate all the grads.
  • Why can’t gradients be zeroed when loss.backward() is called?
    Ans: If so, if we process optimizer.step() after loss.backward(), the gradients will be hidden in the loss.backward() step. But sometimes we need to accumulate gradients among some batches. Refdanche in stackoverflow.

Gradient vanishes problem

  • When and why does this problem happen?
    Ans: If the activation function take input in large range and generate output in a small range (e.g tanh or sigmoid), large change of input may not change the output significantly, this leads to “Gradient vanish”. And this becomes much worse if we stack several layers. The data need to go through every layers. For instance, first layer will map a large input region to a smaller output region, which will be mapped to an even smaller region by the second layer, which will be mapped to an even smaller region by the third layer and so on. RefNikhil Garg in quora.
  • How to avoid this problem?
    Ans: Use activation functions which don’t have this property of ‘squashing’ the input space into a small region. Like RELU, which maps $x$ to $\max (0,x)$.

Add sitemap to make your hexo blog searched through google

Posted on 2017-09-28

I found that I can’t reach the articles in my blog no matter what keywords I choose. After a few minutes digging, I realized that I need to let google know that I build a blog. I follow the instructions in pengbinlee‘s article. Steps are very clear in that site. I did the first 3 steps. Below is what I did:

  1. Add sitemap

    • install hexo-generator-sitemap
      npm install hexo-generator-sitemap --save
    • add sitemap info to hexo’s __config.yml_

      sitemap:
         path: sitemap.xml

  2. Add robots.txt to source

    • new a robots.txt in source directory

      User-agent: *
      Allow: /
      Allow: /archives/
      Allow: /tags/
      Allow: /about/
      Disallow: /vendors/
      Disallow: /js/
      Disallow: /css/
      Disallow: /fonts/
      Disallow: /vendors/
      Disallow: /fancybox/
      Sitemap: https://binglixyz.github.io/sitemap.xml
      Sitemap: https://binglixyz.github.io/baidusitemap.xml

  3. Upload to google through Google Search Console

    • verified your site (add the meta into themes\next\layout\_partials\head.swig if needed)
    • test robots.txt file
    • add sitemap and click Request indexing

Now articles could be searched through google!
